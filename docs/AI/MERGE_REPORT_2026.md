see duplicates below.

## Colleges added to the catalog — 0 (by design)

Unmatched rows are **not** auto-added, so the recommendation input set stays exactly
324. Injecting them would create half-populated colleges (no `nirfId` → no link to
placement/faculty/finance facts) and silently change what recommendation sees. They
are preserved as `warehouse.nirf2026.profiles` and listed below for a future decision.

## Duplicate candidates — 36 (de-duplicated; kept first)

The dataset lists ~36 colleges **twice** — once in the NIRF-coded ranked section and
again in the alphabetical section. The merge matched the NIRF-coded row first and
flagged the later name-matched row as a duplicate (kept the richer NIRF profile).
This is why the "name" method both matches and produces duplicates. Sample:

| Dup row | College | Matched via | Kept row |
|---:|---|---|---:|
| 206 | AAA College of Engineering and Technology | name | 186 |
| 207 | Aalim Muhammed Salegh College of Engineering | name | 145 |
| 209 | Adhiparasakthi College of Engineering | name | 115 |
| 210 | Adithya Institute of Technology (Autonomous) | name | 86 |
| 232 | Arulmurugan College of Engineering | name | 85 |
| 239 | Bethlahem Institute of Engineering | name | 154 |

⚠️ **Review item:** name-based duplicates should be spot-checked — an exact
`comparisonKey` match *could* collide two genuinely different colleges. None injected
duplicate colleges (guaranteed), but the *attribution* of a name match should be
confirmed before this data drives eligibility/ranking.
